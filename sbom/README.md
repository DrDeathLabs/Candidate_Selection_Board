# Software Bill of Materials

This directory contains the Software Bill of Materials (SBOM) for Candidate Selection Board in [CycloneDX](https://cyclonedx.org/) JSON format (spec 1.6).

The SBOM is **image-based**: it is generated by scanning the built container images, so it captures exact pinned dependency versions *and* the operating-system packages inside each container — not just the declared manifest entries.

## Files

| File | Contents |
| --- | --- |
| `selection-board.cdx.json` | Merged SBOM for the entire platform — all components below in one document |
| `backend.cdx.json` | First-party backend image (also runs as worker, parser, ai-gateway, export-service, audit-service) |
| `frontend.cdx.json` | First-party frontend — the application npm dependency tree from `package-lock.json` (react, axios, …) merged with the runtime image's OS/`serve` packages |
| `ocr.cdx.json` | First-party OCR image (Python + Tesseract) |
| `nginx.cdx.json` | `nginx:1.27-alpine` (reverse proxy) |
| `postgres.cdx.json` | `postgres:16-alpine` |
| `redis.cdx.json` | `redis:7-alpine` |
| `minio.cdx.json` | `minio/minio:RELEASE.2025-01-20T14-49-07Z` |
| `opensearch.cdx.json` | `opensearchproject/opensearch:2.17.1` |
| `clamav.cdx.json` | `clamav/clamav:1.4` |
| `thirdparty-images.txt` | The pinned third-party image references, as used by `docker-compose.yml` |

## Regenerating

```bash
bash scripts/generate-sbom.sh
```

Requirements: Docker. The script builds the three first-party images, pulls the six pinned third-party images, scans each with [Syft](https://github.com/anchore/syft) (run via its official container), and merges the results with [cyclonedx-cli](https://github.com/CycloneDX/cyclonedx-cli).

## Automation

CI ([`.github/workflows/ci.yml`](../.github/workflows/ci.yml)) regenerates the SBOM on every push and uploads it as a build artifact. On a tagged release, the merged SBOM and the third-party image manifest are attached to the GitHub Release.

Regenerate and commit the refreshed SBOM whenever dependencies change (see [`CONTRIBUTING.md`](../CONTRIBUTING.md)).
