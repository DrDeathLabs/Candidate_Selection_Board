# Support

Start with the documentation before opening an issue:

- [User Guide](docs/USER_GUIDE.md) — roles, workflows, and admin operations
- [Production Readiness](docs/PRODUCTION.md) — hardening checklist for deployed instances
- [Security Policy](SECURITY.md) — authentication architecture and vulnerability reporting

## Reporting Issues

Open a GitHub issue for bugs, unexpected behavior, or documentation gaps.

Before opening an issue:

1. Check the [Installation troubleshooting](docs/INSTALLATION.md) section.
2. Check existing open and closed issues.
3. Include Docker version, OS, and the relevant container logs (`docker compose logs api --tail=50`).

Do not include real personnel data, credentials, or classified information in issue reports.

## Security Vulnerabilities

Do not report security vulnerabilities in public issues. Follow the process in [SECURITY.md](SECURITY.md) and use GitHub private vulnerability reporting for this repository.
